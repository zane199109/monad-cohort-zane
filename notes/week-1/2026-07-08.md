### 1. Web3 实习手册 — 安全与合规

**链接：** https://web3intern.xyz/zh/security/

#### 核心要点

- **私钥即资产**：Web3 安全的第一原则。私钥泄露等于资产归零，没有"找回密码"这一说
- **助记词是钱包的根密钥**：BIP-39 标准将私钥编码为 12/24 个单词，必须离线存储，绝不上云、不截图、不提交到任何版本控制系统
- **隔离原则**：实验/学习钱包与主力钱包严格分离。频繁与未知合约交互、点击测试网链接本身就是高风险行为
- **社交工程攻击是最大威胁**：钓鱼链接、虚假空投、假冒客服、伪造的 MetaMask 官网

#### 安全存储方案对比

| 方案 | 安全性 | 实操难度 | 适用场景 |
|------|--------|----------|----------|
| 手写助记词 + 保险箱 | ⭐⭐⭐⭐⭐ | 低 | 长期存储，推荐 |
| 硬件钱包（Ledger/Trezor） | ⭐⭐⭐⭐⭐ | 中 | 大额资产，私钥永不离开设备 |
| 密码管理器加密存储 | ⭐⭐⭐ | 低 | 次选方案，确保主密码足够强 |
| 手机备忘录/截图 | ⭐ | 极低 | ❌ 绝对禁止 |

#### 自己的理解

Web3 的安全模型和传统互联网完全不同。传统互联网有"找回密码"机制，有客服，有银行风控——出了问题可以申诉。Web3 没有这些。私钥就是一切，丢了就是丢了。

这带来的是一种全新的责任模式：你必须自己为自己的资产安全负责。没有安全团队帮你兜底。所以建立安全习惯比学会任何技术都重要——隔离钱包、离线备份、谨慎签名，这些是 Web3 开发者的基本功。

### 2. BuildAnything Freshman — 加密钱包

**链接：** https://buildanything.so/zh/tracks/freshman

#### 核心概念

**钱包不是存钱的，是管钥匙的。**
- 很多人误以为钱包像支付宝/微信一样"存着钱"。不对。钱包只做一件事：**管理你的密钥对**
- **公钥 = 地址**：可以公开，用来接收资金。别人知道你的地址没有任何风险
- **私钥 = 签名凭证**：必须绝对保密。有了私钥就拥有了该地址下所有资产的控制权
- **助记词 = 私钥的人类可读备份**：12 个单词就能恢复整个钱包

#### EVM 钱包的工作原理

**密钥推导流程：**
随机数 → 256位私钥 → secp256k1 椭圆曲线 → 公钥 → Keccak-256 哈希 → 取后20字节 → 0x地址

1. **密钥生成**：随机数生成器产生 256 位的私钥（2^256 种可能，几乎不可能碰撞）
2. **公钥推导**：通过椭圆曲线加密（secp256k1）从私钥**单向**推导出公钥（不可逆）
3. **地址生成**：对公钥进行 Keccak-256 哈希，取最后 20 字节，加上 0x 前缀
4. **交易签名**：使用私钥对交易数据进行 ECDSA 签名
5. **交易广播**：将签名后的交易发送到区块链网络的 mempool

#### 钱包类型对比

| 类型 | 安全性 | 便利性 | 适用场景 |
|------|--------|--------|----------|
| 硬件钱包 | ⭐⭐⭐⭐⭐ | ⭐⭐ | 大额资产长期存储 |
| 浏览器插件（MetaMask） | ⭐⭐⭐ | ⭐⭐⭐⭐ | 日常 dApp 交互 |
| 手机 App（Rabby Wallet） | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ | 移动端使用 |
| 纸钱包 | ⭐⭐⭐⭐⭐ | ⭐ | 冷存储备份 |

**为什么选 MetaMask：** EVM 生态最广泛支持，浏览器插件与 dApp 无缝集成，支持自定义网络（可以手动添加任何 EVM 兼容链），开源代码经社区审计。

### 3. Monad 和它的 Testnet

**参考：**
- Monad Docs：https://docs.monad.xyz/
- Testnets：https://docs.monad.xyz/developer-essentials/testnets
- Block Explorers：https://docs.monad.xyz/tooling-and-infra/block-explorers

#### Testnet 是什么

Testnet 是主网（Mainnet）的"沙盒镜像"，具有以下特点：

| 特性 | 说明 |
|------|------|
| 相同的 EVM 环境 | 工具链、合约语言、交易格式与主网完全一致 |
| 免费测试币 | 通过水龙头（Faucet）领取，无真实经济价值 |
| 快速出块 | 测试网通常更快，适合快速迭代 |
| 可重置 | 测试网状态可以重置，不影响主网 |

**关键理解：** 测试币没有经济价值，但有**学习价值**。在 Testnet 上犯的错只浪费测试币，在主网上犯的错损失真金白银。

#### Monad Testnet 参数

| 参数 | 值 |
|------|-----|
| 网络名称 | Monad Testnet |
| RPC URL | https://testnet-rpc.monad.xyz/ |
| Chain ID | 10143 (0x279f) |
| 币种符号 | MONAD |
| 区块浏览器 | https://testnet.monadexplorer.com/ |

#### Monad Testnet 的特殊之处

Monad 是基于 EVM 的高性能 L1 链，目标是 >10,000 TPS。Testnet 目前已有超过 **4300 万区块**，说明其出块速度非常快。这对于开发者意味着：
- 交易确认时间极短（秒级）
- 测试迭代效率高
- 但也意味着链上数据量快速增长

### 4. 交易与 Gas 机制

**参考：**
- Ethereum Transactions：https://ethereum.org/en/developers/docs/transactions/
- Ethereum Gas：https://ethereum.org/en/developers/docs/gas/
- Monad Gas Pricing：https://docs.monad.xyz/developer-essentials/gas-pricing

#### 交易的结构

一笔 EVM 交易包含以下字段：

| 字段 | 说明 | 我的交易中的值 |
|------|------|---------------|
| **from** | 发送方地址 | `0x8EB3...96BE2C` |
| **to** | 接收方地址（合约部署则为 null） | `0xc2dc...3c9d6` |
| **value** | 转账金额（wei 为单位） | 1 MONAD |
| **data/input** | 合约调用数据（转账则为空） | 0x |
| **gas limit** | 允许消耗的最大 gas | 21,000 |
| **maxFeePerGas** | EIP-1559 最大费用上限 | 138.45 Gwei |
| **maxPriorityFeePerGas** | 验证者小费 | 3.45 Gwei |
| **nonce** | 发送方的交易序号 | 0（第一笔） |
| **v, r, s** | ECDSA 签名参数 | — |

#### Gas 机制详解

**Gas = 链上计算资源的计价单位。**

> 总手续费 = gasUsed × effectiveGasPrice

- **基础 Gas（21,000）**：每笔简单 EOA 转账固定的 gas 消耗量
- **合约交互**：调用智能合约会消耗更多 gas（取决于计算复杂度）
- **Gas 价格**：由网络供需决定，拥堵时上涨、空闲时下降

**EIP-1559 费用模型（type 0x2）：**
> effectiveGasPrice = baseFee + priorityFee

- **baseFee（基础费）**：由协议算法算出，根据网络拥堵程度自动调整，**全部销毁**
- **priorityFee（优先费/小费）**：用户自愿给验证者的激励，出价越高优先被打包

**这次交易的费用明细：**
> 21,000 gas × 103.45 Gwei = 0.002172 MONAD
> 其中 baseFee ≈ 100 Gwei（销毁），priorityFee ≈ 3.45 Gwei（给验证者）

#### 为什么失败交易也消耗 Gas

这是很多新手困惑的点。原因是：

> **节点已经执行了计算。即使交易 revert，计算资源已经消耗了。**

好比你去餐厅点了菜，厨房已经开始做了，但做到一半发现食材不够做不了。你仍然要为已经发生的劳动付费。在链上：
- 已执行的 **gasUsed** 不退（已经消耗的算力）
- 但未用完的 **gas limit** 余额会退（相当于预存的押金没花完的部分）

这也解释了为什么要把 gas limit 设得比预期稍高一些——如果设得太低，交易会因为 out-of-gas 而失败，gas 已经花了但交易没成功。

## 关键收获

1. **钱包本质是密钥管理工具**，不是"电子钱包"。私钥泄露 = 资产归零，没有找回机制
2. **隔离原则**是 Web3 安全的基本纪律。实验钱包与主力钱包严格分离
3. **助记词必须离线手写备份**，任何电子化存储都是安全隐患
4. **Testnet 是安全的练手环境**，配置参数必须准确（Chain ID 10143）
5. **交易生命周期**：签名 → 广播 → Mempool → 打包 → 出块 → 确认
6. **Gas 机制**：基础转账 21,000 gas，EIP-1559 下有 base fee（销毁）+ priority fee（小费）
7. **失败交易也消耗 Gas**：计算资源已消耗，已用的 Gas 不退，未用完的 limit 退
8. **Nonce 从 0 开始计数**，标识该地址发起的每笔交易，防止双花
9. **Faucet 是获取测试币的主要方式**：https://faucet.monad.xyz/
10. **区块浏览器是链上的"搜索引擎"**，可查地址/交易/合约/网络状态
