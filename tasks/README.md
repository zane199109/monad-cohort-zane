# 每周任务

任务按轨道分类，每周在对应轨道目录下创建 `week-N/` 子目录，完成该周任务。

## 轨道分类

| 轨道 | 说明 |
|------|------|
| `general/` | 通用任务：常规任务 |
| `technical/` | 技术任务：编码、合约、Indexer、工具链 |
| `operations/` | 运营任务：社区管理、内容发布、推广 |
| `research/` | 研究任务：深度研究、论文阅读、竞品分析 |

## 每周任务结构

```
tasks/
├── general/
│   ├── week-1/
│   │   └── TASKS.md
│   └── week-2/
│       └── TASKS.md
├── technical/
│   ├── week-1/
│   │   └── TASKS.md
│   └── ...
├── operations/
│   ├── week-1/
│   │   └── TASKS.md
│   └── ...
└── research/
    ├── week-1/
    │   └── TASKS.md
    └── ...
```

## 任务文件格式

每个 `week-N/TASKS.md` 使用以下格式：

```markdown
## Week N Tasks

### 目标
[本周轨道学习目标]

### 任务清单
- [ ] 任务描述 — 截止：Day X

### 完成记录
- [x] 已完成任务 + 链接/截图

### 遇到问题
- [问题描述] → [解决过程]
```
